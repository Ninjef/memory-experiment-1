# FIRST_STEP Implementation — 2026-03-31

## What was built
A modular memory processing pipeline per FIRST_STEP.md requirements: ingest text chunks, cluster them, and use an LLM to generate higher-level insights.

## Architecture
Three-stage pipeline using Python `typing.Protocol` for swappability:

1. **Embedder** (`src/embedder.py`) — Local `sentence-transformers` model (`all-MiniLM-L6-v2`, 384-dim). No API cost.
2. **Clusterer** (`src/clusterer.py`) — UMAP dimensionality reduction → HDBSCAN clustering.
3. **Synthesizer** (`src/synthesizer.py`) — Anthropic API calls (one per cluster) to generate 1-3 insights per cluster.

Orchestrated by `Pipeline` in `src/pipeline.py`, which returns a `PipelineResult` containing insights, clusters, and input chunks.

## Data format
- **Input:** JSONL — each line has required `text` field plus arbitrary metadata (timestamps, source, speaker, etc.)
- **Output:** A run folder (`output/run_<timestamp>/`) containing:
  - `insights.json` — pretty-printed insights with source memory texts inline
  - `clusters.json` — cluster breakdown with all member texts and metadata
  - `run_info.json` — config, stats, and cluster size information

## Key decisions
| Decision | Choice | Rationale |
|---|---|---|
| Data format | JSONL | Flexible metadata without column explosion, human-readable |
| Embeddings | Local sentence-transformers | Free, fast, no API cost; produces vectors needed for future Zero-Shot work |
| Clustering | UMAP + HDBSCAN | Industry standard; directly needed for IDEA_OVERVIEW's future approach |
| Abstractions | Protocol classes | Lighter than ABC; any object with the right method works |
| Output | Pretty JSON in run folders | Human-readable; includes source texts, cluster details, and run metadata |

## How to run
```bash
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env  # add ANTHROPIC_API_KEY
python run.py data/sample_input.jsonl        # basic run
python run.py data/sample_input.jsonl -v     # verbose logging
```
