# 3D Cluster Visualization — 2026-04-01

## What was built
Interactive 3D browser visualization of cluster results. Points represent memory chunks positioned by UMAP 3D reduction, color-coded by cluster assignment. Rotatable, zoomable, with hover tooltips.

## Changes made

### New file
- `src/visualizer.py` — Generates a self-contained HTML file using Three.js (CDN) with OrbitControls, raycaster hover tooltips, cluster legend, and all data embedded inline as JSON.

### Modified files
| File | Change |
|---|---|
| `src/clusterer.py` | Added `reduce_for_viz(chunks, n_components=3)` — runs a separate UMAP pass (384→3 dims) independent from the clustering UMAP (384→10 dims) |
| `src/pipeline.py` | Added `viz_coords` field to `PipelineResult` (dict mapping chunk ID → (x,y,z)). Added `_build_viz_coords()` helper called in both `run()` and `run_cluster_only()` |
| `src/io.py` | Added `_save_viz_coords()` writing CSV (id, cluster_id, x, y, z, text). `save_run()` now also calls `generate_viz_html()` when viz coords exist |
| `run.py` | Prints `viz_coords.csv` and `viz.html` paths in output summary |

## Output additions
Run folders now include two new files:
- `viz_coords.csv` — id, cluster_id, x, y, z, text (machine-readable)
- `viz.html` — open directly in browser, no server needed

## Key decisions
| Decision | Choice | Rationale |
|---|---|---|
| Separate UMAP pass | 3-dim viz UMAP independent from 10-dim clustering UMAP | Keeps clustering quality unaffected; 3D is for human viewing only |
| Data storage | CSV for coordinates, inline JSON in HTML | CSV is portable/parseable; inline JSON makes HTML fully self-contained |
| Viz library | Three.js via CDN importmap | No build step, no npm; single HTML file works offline after first CDN fetch |
| Point rendering | MeshStandardMaterial spheres with emissive glow | Visually distinct, hover highlight via scale+emissive boost |

## Visualization features
- 3D scatter plot with OrbitControls (rotate, zoom, pan)
- Color-coded clusters (10-color palette, noise in gray)
- Hover tooltips showing cluster label and memory text
- Legend with cluster names and member counts
- Responsive resize, dark background

## How to use
```bash
python run.py data/sample_input.jsonl --cluster-only -v
open output/run_<timestamp>/viz.html
```
