# Output UX Improvements — 2026-04-02

## What was built
Three quality-of-life improvements to make experiment runs easier to identify and review:

1. **Fun names for output run folders** — folders now get random two-word names like `run_20260402_174352_breezy-cobalt` instead of bare timestamps
2. **`cluster_texts.json`** — a new minimal output file showing just the text per cluster, sorted by timestamp, with an empty `insights` list ready for future use
3. **Extraction script folder structure** — `extract_locomo.py` now creates a named run folder with `run_info.json` for each extraction, mirroring the pipeline output pattern

## Files created
```
src/naming.py   — Word lists (50 adjectives × 50 nouns = 2,500 combos) and
                  generate_run_name() which picks a random pair and handles
                  collisions by appending _2, _3, etc.
```

## Files modified
```
run.py                                  — Imports generate_run_name(), uses it for
                                          default output folder names; added
                                          cluster_texts.json to console output summary
src/io.py                               — Added _save_cluster_texts() function that
                                          writes per-cluster objects with "insights" (empty
                                          list) and "cluster_texts" (texts sorted by
                                          timestamp); added "run_name" field to run_info.json;
                                          wired cluster_texts.json into save_run()
rawDataPreFormatted/extract_locomo.py   — Restructured to create a timestamped run folder
                                          with fun name per extraction; added
                                          save_extract_info() that writes run_info.json
                                          capturing all input parameters (input file, mode,
                                          limit, sample, seed, file list, total records);
                                          simplified write_jsonl() since folder creation
                                          is now handled separately
```

## Key decisions
| Decision | Choice | Rationale |
|---|---|---|
| Name collision handling | Append `_2`, `_3` etc. | Simple, predictable; timestamps already make collisions rare |
| Word list size | 50 × 50 | 2,500 combos is plenty for uniqueness without a huge list to maintain |
| Shared naming module | `src/naming.py` imported by both `run.py` and `extract_locomo.py` | Single source of truth for name generation |
| cluster_texts.json structure | Object per cluster with `insights` + `cluster_texts` | Prepares for future insight population while keeping the minimal text view useful now |
| Timestamp sorting | Sort by `metadata["timestamp"]`, empty strings sort first | Chunks without timestamps float to top rather than causing errors |
| Extraction run_info.json | Captures all CLI params including nulls | Makes it unambiguous what defaults were used vs what was explicitly set |
