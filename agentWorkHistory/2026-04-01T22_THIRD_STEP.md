# THIRD_STEP Implementation — 2026-04-01

## What was built
A pluggable steering system that implements step 4 of the "Zero-Shot Latent Space Steering" idea from IDEA_OVERVIEW.md. Steering distorts memory embeddings to amplify specific themes before clustering, forcing previously unrelated memories to cluster together. The system is modular: each steering approach lives in its own file and is loaded dynamically at runtime.

## Architecture

The pipeline now flows: `embed → (steer) → cluster → synthesize`

Steering is optional — omitting `--steerer` preserves the original behavior. Each steerer module is a standalone Python file in `src/steerers/` that exposes:

- A class implementing `steer(chunks) -> chunks`
- `add_args(parser)` — registers steerer-specific CLI flags
- `create(args, embedder)` — factory that builds the steerer from parsed args

Dynamic module loading via `importlib` means adding a new steerer is just dropping a new `.py` file in `src/steerers/`.

### Two-phase argparse

The CLI uses a two-phase parse to support per-steerer arguments without breaking modularity:

1. A lightweight pre-parser (no `--help`) extracts just `--steerer`
2. If a steerer is specified, its module is loaded and `add_args(parser)` registers steerer-specific flags on the main parser
3. The main parser then runs `parse_args()` with full help support

This means `python run.py --steerer projection --help` shows both base and projection-specific args.

### Dual visualization

When steering is active, the pipeline outputs two visualizations:
- `viz_steered.html` — the distorted embedding space (what clustering actually saw)
- `viz_original.html` — the original embedding space, but with cluster colors from the steered clustering

This lets you visually compare how steering pulled previously distant memories together.

## First steerer: Projection (`src/steerers/projection.py`)

Implements steps 3-4 from IDEA_OVERVIEW. For each theme vector `t` and memory embedding `m`:

1. `proj = (m · t / t · t) * t` — component of m along the theme direction
2. `steered = m + (alpha - 1) * proj` — amplify that component

Themes are embedded using the same sentence-transformers model as the memories, ensuring they live in the same vector space. Multiple themes are applied sequentially.

### CLI arguments
| Flag | Default | Purpose |
|---|---|---|
| `--themes` | — | Theme strings inline, e.g. `--themes "financial anxieties" "food and budget"` |
| `--themes-file` | — | Path to newline-delimited theme file |
| `--steer-alpha` | 2.0 | Amplification factor for theme-aligned components |

## Key decisions
| Decision | Choice | Rationale |
|---|---|---|
| Plugin discovery | `importlib.import_module` by name | Simple, no registry needed — drop a file, use it |
| Per-steerer args | Two-phase argparse with `add_args`/`create` convention | Keeps CLI modular; steerer args only appear when steerer is selected |
| Embedding snapshots | `copy.deepcopy` before steering | Steerer mutates embeddings in-place; need originals for dual viz |
| Dual viz | Separate HTML files, not a toggle in one file | Simpler, can open side-by-side in browser tabs |
| Clustering config | `--clusterer` flag as placeholder (only `umap_hdbscan`) | Infrastructure ready for same plugin pattern when a second algorithm is needed |
| Theme embedding | Reuse the pipeline's embedder instance | Themes must be in the same vector space; avoids loading the model twice |

## How to run
```bash
# Without steering (backward-compatible)
python run.py data/sample_input.jsonl --cluster-only

# With projection steering
python run.py data/sample_input.jsonl --steerer projection --themes "financial anxieties" "food and budget" --cluster-only

# Custom alpha, themes from file
python run.py data/sample_input.jsonl --steerer projection --themes-file themes.txt --steer-alpha 3.0 --cluster-only

# See steerer-specific help
python run.py --steerer projection --help
```

## Adding a new steerer
1. Create `src/steerers/my_approach.py`
2. Implement a class with `steer(chunks) -> chunks`
3. Add module-level `add_args(parser)` and `create(args, embedder)` functions
4. Use it: `python run.py input.jsonl --steerer my_approach`
